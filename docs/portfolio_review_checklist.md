# Portfolio review checklist (for a reviewer)

This checklist is a fast navigation guide.

## If you have 5 minutes

1) Read `README.md` (overview + highlighted projects).  
2) Open `PROJECTS.md` (projects table and statuses).

## If you have 15 minutes

1) Read `README.md`.  
2) Read `docs/Mapping_to_MS_prerequisites.md`.  
3) Open the most recent completed project repo (term project) and read its `docs/report.md`.

## If you have 30+ minutes

1) Skim `ROADMAP.md` for the curriculum structure.
2) Read one full technical report from a term project.
3) Review the repository structure, tests, and reproducibility commands.

## Evidence standards (what each project should include)

- Clear problem statement and dataset description.
- Methods section with modelling choices and evaluation design.
- Results section with quantitative metrics and discussion.
- Responsible AI: limitations, risks, mitigation and monitoring.
- Reproducible scripts + minimal tests.
